【爬虫】使用python爬取github上的代码
更新于 2020-12-08 技术 热度 1 喜欢 0

最近在做毕设，需要训练一个代码补全的深度学习模型，其中训练模型的数据是python代码，因此需要用爬虫来爬取github上的代码数据。

整体思路

确定API

不同于其他大型网站，github提供了专属的api来用于网站的数据查询和获取，而且没有增加比较复杂的反爬虫机制，因此在爬取github网站上的数据时会非常容易。

API链接和相关文档如下：

  • GitHub REST API
  • GitHub API v3 概述

我的任务是根据关键词（比如”tensorflow”）来爬取github仓库中的代码，同时根据仓库的stars数逆序排序结果，可直接可使用相应的v3接口，即可查看api返回的查询结果，例如：

https://api.github.com/search/repositories?o=desc&q=tensorflow&sort=stars

默认返回30个仓库的结果，后续如果我想爬取海量的tensorflow代码仓库，直接增加关键字page和pre_page，例如：

https://api.github.com/search/repositories?o=desc&q=tensorflow&sort=stars&page=i&pre_page=100
可以根据for循环从1开始更新i的结果，pre_page为每页展示多少个代码仓库，最大设置数为100，在实际爬取过程中，pre_page的值貌似最大才50多。
【注】在不理解参数的实际含义时，api参数最好不要写缩写，我当时以为page的缩写是p，在使用过程中一直用的p，导致当时的结果一直有问题，因为在循环过程中p的值一直在变，但是返回的页面结果始终是一样的，我最开始以为是api的问题，也因此被卡了很久，看文档也没找到问题，实际上p貌似没有含义，应该用page!

进行登录验证

因为github网站的访问限制，所以为了爬取海量的数据，需要先进行登录验证，保存用户session和授权码token，因此这一步骤主要是为了正常登录所进行的一系列参数获取及设置。
